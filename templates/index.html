Passo 1 — (Opcional) Completar dados da Kitchen

Auto-cadastro via CSV

Ao importar o CSV, o sistema cria automaticamente kitchens novas (você só completa nome/CNPJ se quiser).

Passo 2 — Lançar Estoque

INITIAL / compra / ajuste
Regra importante: se a kitchen não tiver INITIAL, ela fica em ALERTA com saldo 0.
Quando usar cada tipo?
  • INITIAL: o saldo que vale a partir do go-live (uma vez por kitchen; pode refazer lançando outro INITIAL mais recente).
  • PURCHASE: novas compras de etiquetas (somar no saldo).
  • ADJUST: correção/estorno (pode ser negativo).

Passo 3 — Importar Consumo (CSV do Metabase)

Recomendado: 1x/dia
Colunas obrigatórias: kitchen_id, day (YYYY-MM-DD), labels_used.
Dica: exporte pelo menos os últimos 120 dias para a média de 90 dias funcionar bem. O CSV pode conter todas as kitchens — o site cadastra automaticamente.
Query sugerida no Metabase (copiar/colar) 
SELECT
  l.kitchen_id,
  to_char(date_trunc('day', lp.created_at)::date, 'YYYY-MM-DD') AS day,
  COUNT(*)::int AS labels_used
FROM labelprint lp
JOIN label l ON l.id = lp.label_id
WHERE 1=1
  [[ AND lp.created_at >= {{start}} ]]
  [[ AND lp.created_at <  {{end}} ]]
GROUP BY 1, 2
ORDER BY day DESC;
